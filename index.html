Academic Projects

USA Election Contribution Analysis

This project focused on analyzing large-scale U.S. election contribution data using two data architectures: a relational data warehouse and Hadoop.

  • I designed a relational schema with clear table relationships, data dictionary, and sample values.
  • Built ETL pipelines for both systems to clean, transform, and load data.
  • Used SQL Server for the warehouse and Hive/Spark for the Hadoop implementation.
  • Developed reports to compare how both systems handled the same analysis tasks in terms of performance, scalability, and usability.
  • Reflected on the pros and cons of each approach when managing large public datasets.
  • Delivered visual dashboards and SQL/Hive queries to extract political donation patterns across candidates, regions, and time.

    • Key Skills:

    SQL, Hadoop, ETL, Data Warehousing, Spark, Hive, Performance Analysis

The Intersection of Education and Opioid Misuse: A Data-Driven Analysis

This research-driven project examined how education levels correlate with opioid misuse across different demographics in the U.S.

  • Defined the problem, conducted initial research, and framed business/research questions.
  • Collected public datasets, performed data cleaning, and created a data dictionary with labeled variables.
  • Explored cognitive biases that could influence data interpretation.
  • Built descriptive charts (e.g., age of first opioid use by education level, substance initiation by race) and GIS maps showing regional trends.
  • Developed predictive models using Python to assess risk factors and generate insights.
  • Compared descriptive, predictive, and prescriptive findings, and discussed strategies to mitigate overfitting and bias.
  • Summarized key insights for public health and education policy impact.

    • Key Skills:

    Python, Data Cleaning, Visualization, Predictive Modeling
